# RJP Codes Style

2019年08月17日10:28:39

今天在写代码的过程中

发现了自己的一些小问题, 做个记录



- 函数输入输出参数命名过长

  解决方案: 在声明时可以长一些, 但在实现时, 必须精简

  变量注释可以在声明中写清楚, 然后用的时候尽量简短

- 函数注释写法

  谷歌 C++ 规范, 在声明处的注释里描述函数的功能, 在实现处描述函数实现的具体做法

  此处, 如何描述函数功能呢? 是具体到对变量进行的操作还是单纯从逻辑上进行描述

- 权衡函数输入的传值或传址

  多于比较大的输入变量, 通过 const 修饰传址, 可以保证在函数内部不被修改, 同时节省内存

  但对于已知的小变量也需要这么做吗? 或者小变量值得这么做吗?

- 简短重复的功能是否应该写成一个函数

- 如何区分返回 bool 的函数, 其 bool 返回值是单纯判断函数是否执行完毕或有实际意义
